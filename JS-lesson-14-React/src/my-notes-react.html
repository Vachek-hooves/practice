Види компонентів: Functional Class- компонента обов'язково мусить містити імпорт реакту. (классова компонента описується так: першочергово імпортується react.) import React from 'react'; - React - з великої літери пишеться завжди коли імпортується. 'react'- вказується папа node_modules. class NewsItemClass extends React.Component; - коли описується классова компонента, створюється класс який наз так як і наша компонента (NewsItemClass), вона обов'язково має наслідувати весь функціонал який є в реакті в компоненті. Тому після назви классу пишеться extends і звертається до реакту, що імортували до його об'єкту components- (React.Component) render()- метод щоб вивести верстку в классовій компоненті, а в ньому вся інша логіка.

Для того щоб компоненту зробити універсальною, не варто в ній прописувати верстку і в ній додаткові тексти. Компонента має бути універсальною, і отримувати тексти від свого батька (куди компонента додається там і отримує тексти).

Props (пропси)- це вхідні дані React-компонента, які призначенні для читання і не можуть змінені. Вони є даними, що передаються від батьківського компонента до дочірнього. В середині компонента (functional або Class) є змінна props. Вона є об'єктом всередині в якої додаються всі додаткові змінні. Props - передається в місці де підключається компонента і моєе бути будь-як названа. NewsItemFunc title="" text="" / Доступ до пропси знаходяться всередині верстки. Props - оголошуютсья в функції (передається як параметр). !!!!!!! Данні в компоненті статично не прописуються, вони всі оголошуютсья динамічно. Props через классову компоненту пишуться так само як і в функціональний, але через this. Перед цим потрібно прописати конструкторб в нього прописати props, потім викликати метод super і туди також передається props.

STATEFUL AND STATELESS COMPONENTS. STATELESS COMPONENTS - (функціональні компоненти), компонент для виведення даних. Коли потрібно просто показати props, коли не потрібен будь-який стан чи змінна, коли не потрібний багаторазовий код. презентаційна компонента котра ніяк не взаємодіє з State. Є тільки виведення даних на основі вхідних данних (на основі props). STATEFUL COMPONENTS - (классова компонента), приймає данні, опрацьовує їх і повертає якийсь результат. ЦЕ в більшості соїй має state і з ним працює. Зазвичай вони створюються на основі react component. Приклад :Couter.

Event -поняття івенту в реакті не змінюєтьсял, крім додавання слухачів event. В React це поняття змінилося, події як в js лишаються, крім DOMContentLoaded (в ній сенсу немає), лишається перевірка,чи компонент був вмонтований.

Якщо потрібно додати слухачі до window (ширину або висоту екрану) використовується - ComponentDidMount.

Event в реакті додаються через "on" (onClick, onChange, onFocus, onLoaded)...

Слухачі в React додаються одразу до елементу до якого буде відстежуватися подія. Якщо методів буде декілька, тоді вони додаються через крапка з кому. Метод в реакт не рахується інлайновим. this - застосовується до звернення state або props та методу. Коли знаходиться функція поза классом, тоді можна звертатися без this.
